1. IDENTIFIKASI CORE MODULES/SERVICES
----------------------------

Berdasarkan analisis domain, MediTrack didekomposisi menjadi 6 Bounded Contexts:


SERVICE 1: USER & IDENTITY MANAGEMENT SERVICE
──────────────────────────────────────────────

Tanggung Jawab:
• Authentication & authorization
• User profile management
• Role-based access control (RBAC)
• Permission management
• Audit logging untuk security events

Key Classes/Entities:
- User
  * Properties: id, name, email, phone, address, role, status
  * Methods: authenticate(), updateProfile(), changePassword()
  
- Role
  * Properties: id, name, permissions
  * Methods: addPermission(), removePermission()
  
- Permission
  * Properties: id, name, description
  
- AuditLog
  * Properties: id, user_id, action, timestamp, ip_address

Dependencies: None (foundation)
Performance Requirements: Auth harus <100ms


SERVICE 2: APPOINTMENT & SCHEDULING SERVICE
─────────────────────────────────────────────

Tanggung Jawab:
• Appointment booking, rescheduling, cancellation
• Conflict detection dalam jadwal dokter
• Appointment status management
• Notification sending
• Availability checking

Key Classes/Entities:
- Appointment
  * Properties: id, patient_id, doctor_id, appointment_date, status, 
               type, location, duration, description, notes
  * Methods: book(), reschedule(), cancel(), confirm(), getStatus()
  * Validations: No doctor double-booking, valid datetime
  
- AppointmentSchedule
  * Properties: id, doctor_id, day_of_week, start_time, end_time, slots
  * Methods: getAvailableSlots(), isAvailable(), reserveSlot()
  
- AppointmentNotification
  * Properties: id, appointment_id, type, recipient, sent_at
  * Methods: send(), markAsSent()

- Slot
  * Properties: id, schedule_id, start_time, status, booked_by
  * Methods: reserve(), release(), isAvailable()

Key Business Rules:
- Doctor tidak bisa memiliki 2 appointment overlapping
- Appointment minimal 24 jam sebelumnya
- Cancellation harus min 2 jam sebelum appointment

Dependencies: User & Identity Service
Performance Requirements: Availability check <500ms, Book appointment <1s


SERVICE 3: CLINICAL & MEDICAL RECORDS SERVICE
──────────────────────────────────────────────

Tanggung Jawab:
• Medical records storage & retrieval
• Lab results management
• Prescription data
• Clinical history
• Compliance dengan healthcare regulations

Key Classes/Entities:
- MedicalRecord
  * Properties: id, patient_id, doctor_id, appointment_id, 
               diagnosis, treatment, notes, created_at
  * Methods: create(), getHistory(), export(), updateRecord()
  * Access Control: Only doctor & patient
  
- LabResult
  * Properties: id, patient_id, test_type, result_value, 
               reference_range, status, lab_date
  * Methods: upload(), process(), notify()
  
- Prescription
  * Properties: id, patient_id, doctor_id, medication_id, dosage,
               frequency, duration, refills, status
  * Methods: create(), approve(), dispense(), refill()
  
- PrescriptionOrder
  * Properties: id, prescription_id, pharmacy_id, status, 
               fulfillment_date
  * Methods: submit(), accept(), dispense()

- ClinicalNote
  * Properties: id, patient_id, doctor_id, note_type, content, 
               timestamp
  * Methods: create(), edit(), archive()

Dependencies: User, Appointment Service
Performance Requirements: Record retrieval <300ms, History query <1s
Compliance: HIPAA-ready logging, encryption at rest


SERVICE 4: PHARMACY & DRUG STOCK SERVICE
─────────────────────────────────────────

Tanggung Jawab:
• Drug inventory management
• Stock level tracking
• Reorder automation
• Pharmacy location management
• Integration dengan external pharmacy APIs

Key Classes/Entities:
- Pharmacy
  * Properties: id, name, location, phone, manager_id, status
  * Methods: updateStock(), processOrder(), getInventory()
  
- DrugStock
  * Properties: id, pharmacy_id, drug_id, quantity, 
               reorder_level, expiry_date, batch_number
  * Methods: decreaseStock(), increaseStock(), isLowStock(), checkExpiry()
  
- Drug
  * Properties: id, name, generic_name, manufacturer, 
               category, price, description
  * Methods: getStockAcrossPharmacies()
  
- InventoryTransaction
  * Properties: id, drug_id, pharmacy_id, type, quantity, 
               timestamp, reason, reference_id
  * Methods: log()

Key Business Rules:
- Automatic reorder ketika stock < reorder_level
- Warning untuk expired/expiring drugs
- Track stock mutations untuk audit

Dependencies: Prescription Service
Performance Requirements: Stock query <200ms, Reorder <100ms


SERVICE 5: PAYMENT & FINANCIAL SERVICE
───────────────────────────────────────

Tanggung Jawab:
• Payment processing
• Insurance claims
• Invoice generation
• Payment history tracking
• Integration dengan payment gateways
• Refund handling

Key Classes/Entities:
- Payment
  * Properties: id, patient_id, appointment_id, amount, 
               method, status, transaction_id, paid_at
  * Methods: process(), capture(), refund(), getReceipt()
  
- Invoice
  * Properties: id, patient_id, appointment_id, services, amount,
               tax, total, issued_date, due_date, status
  * Methods: generate(), send(), markPaid(), createReminder()
  
- InsuranceClaim
  * Properties: id, patient_id, appointment_id, claim_amount,
               claim_date, status, response_date, approved_amount
  * Methods: submit(), track(), updateStatus(), autoApprove()
  
- PaymentMethod
  * Properties: id, type (credit_card, bank_transfer, etc)
  * Methods: validate(), getProvider()

- Transaction
  * Properties: id, payment_id, gateway_id, status, timestamp
  * Methods: log(), retry()

Dependencies: User, Appointment, Medical Record Services
Performance Requirements: Payment process <2s, claim submission <500ms


SERVICE 6: ANALYTICS & REPORTING SERVICE
──────────────────────────────────────────

Tanggung Jawab:
• Patient outcomes tracking
• Doctor performance metrics
• Drug usage analytics
• System health monitoring
• Business intelligence reporting

Key Classes/Entities:
- PatientOutcomeMetric
  * Properties: id, patient_id, metric_type, value, measurement_date
  * Methods: calculate(), aggregate(), generateReport()
  
- DoctorPerformanceMetric
  * Properties: id, doctor_id, metric_type (completion_rate, 
               satisfaction, efficiency), value
  * Methods: calculate(), benchmark(), trend()
  
- DrugUsageAnalytic
  * Properties: id, drug_id, usage_count, total_quantity, 
               period, trend
  * Methods: track(), compareTrends(), predictDemand()
  
- Report
  * Properties: id, report_type, parameters, generated_at, 
               generated_by
  * Methods: generate(), export(), schedule(), email()
  
- SystemHealthMetric
  * Properties: id, metric_name, value, timestamp, status
  * Methods: collect(), alert(), trend()

Key Business Rules:
- Analytics harus near real-time (max 5 min delay)
- Historical data kept untuk 5 tahun
- Prediction model untuk demand forecasting

Dependencies: Read-only access ke semua services
Performance Requirements: Report generation <5s, metrics <100ms

─────────────────────────────────────────────────────────────────────────────

INTER-SERVICE COMMUNICATION:
──────────────────────────────

Appointment Service
       │
       ├─→ calls: User Identity Service (validate doctor/patient)
       ├─→ calls: Notification Service (send reminders)
       └─→ triggers: Payment Service (when appointment confirmed)
               
Medical Record Service
       │
       ├─→ calls: User Identity Service (access control)
       ├─→ calls: Appointment Service (get appointment data)
       └─→ publishes: Event "RecordCreated" (for analytics)

Pharmacy Service
       │
       ├─→ calls: Prescription Service (get orders)
       ├─→ publishes: Event "StockLow" (for alerts)
       └─→ calls: Notification Service (reorder alerts)

Payment Service
       │
       ├─→ calls: Insurance Service (claim processing)
       ├─→ calls: Appointment Service (get appointment data)
       └─→ publishes: Event "PaymentProcessed" (for audit)

Analytics Service
       │
       └─→ subscribes: ALL services' events (reads from event log)
